# Week 13：第2回レポート課題（プログラム中間提出）

!!! info "Week 13 の位置づけ"
    Week 13 は、第2回レポート課題の進捗確認の週である。  
    ここでは、ゲームの基本機能が動くかを確認し、次の週の完成版に向けて整理する。

## 1. この週の目標

- 1つの簡単なオンラインゲームを作成する
- サーバー側とクライアント側の基本構造を作る
- Week 14 の最終提出に向けて、実装内容を整理する

## 2. 参考ゲーム例

以下のページは、第2回レポート課題で利用できる参考例である。どちらも必須ではない。1つを参考にして改良してもよいし、自分で別のゲームを作成してもよい。

- [RPS ゲームの参考例](report2-rps-example.md)
- [Pictionary ゲームの参考例](report2-pictionary-example.md)

!!! warning "注意"
    補足資料のコードをそのまま提出することは不可である。  
    ルール、デザイン、メッセージ表示、機能などに、何らかのオリジナル要素を加える必要がある。

## 3. 使うフォルダ構成

```text
report2/
├── client/
│   └── src/
└── server/
    └── server.js
```

### 3.1 各フォルダの役割

- `client/src/` では、React で表示する画面やコンポーネントを作る
- `server/server.js` では、Socket.IO を使ってクライアント同士の通信を管理する
- `Week13/` は、提出用にまとめるフォルダである

## 4. 提出物

- Week13/
    - server.js
    - src/
- ZIP 形式で圧縮する
- ファイル名: week13_氏名.zip
- Moodle+R に提出する

### 4.1 提出方法

1. `report2/server/server.js` を `Week13/server.js` にコピーする
2. `report2/client/src` を `Week13/src` にコピーする
3. `Week13` フォルダを ZIP 圧縮する
4. `week13_氏名.zip` という名前で提出する

!!! note "Week 13 の提出について"
    Week 13 では PDF レポートは不要である。  
    ゲームが未完成でもよい。ただし、進捗がわかるように、動作する部分を含めて提出すること。

### 4.2 提出時に意識すること

!!! warning "重要"
    アプリ画面の上部に、名前と学籍番号を表示すること。  
    これが書かれていない場合は、提出物として確認しづらく、評価対象にならないことがある。

- 画面上で自分の作品であることが分かるようにしておくこと
- 変更した部分がわかるように、実装内容を整理しておくこと

- まずは動く最小構成を作る
- サーバー側とクライアント側の役割を分けて考える
- 変更した部分がどこか分かるように整理する

## 5. 進め方の例

- まずはサーバー側で接続を受け付ける
- 次に、クライアント側で画面を作る
- その後、Socket.IO イベントを追加する
- 最後に、簡単なルールや結果表示を入れる
