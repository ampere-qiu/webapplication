    Week 10: Introduction to Socket.IO¶

    このページでは、Socket.IO を用いて、簡単なリアルタイムチャットアプリを作成する。

    Week 10 では、次の4つの Step を順番に実装する。

    • Step 1: Server から React Client にメッセージを送る
    • Step 2: React Client から Server にメッセージを送る
    • Step 3: 匿名チャットを作成する
    • Step 4: 名前付きチャットを作成する

    Important

    Mini-kadai として提出するのは Step 4 の完成版のみ である。

    Step 1〜Step 3 は練習用である。ただし、Step 4 を理解するために順番に実装することを推奨する。

    1. この週で学ぶこと¶

    Week 10 では、リアルタイム通信の基本を学ぶ。

    通常の Web アプリケーションでは、ブラウザが Server にリクエストを送り、Server がレスポンスを返す。

    Browser → Server → Response

    しかし、チャットアプリでは、あるユーザーがメッセージを送ったときに、他のユーザーの画面にもすぐにメッセージを表示する必要がある。

    このような処理には、Server から Client にデータを送る仕組みが必要である。

    この授業では、そのために Socket.IO を使う。

    2. Socket.IO の基本的な考え方¶

    2.1 Socket.IO とは¶

    Socket.IO は、Server と Client の間でリアルタイム通信を行うための JavaScript ライブラリである。

    Socket.IO を使うと、次のような通信ができる。

    • Server から Client にメッセージを送る
    • Client から Server にメッセージを送る
    • Server からすべての Client にメッセージを送る

    2.2 emit と on¶

    Socket.IO では、イベント名を使ってデータを送受信する。

    データを送るときは emit を使う。

    socket.emit('event_name', data);

    データを受け取るときは on を使う。

    socket.on('event_name', (data) => {
  // received data
});

    2.3 socket.emit と io.emit¶

    Socket.IO では、どの相手に送るかによって使うコードが変わる。

    Code Meaning
    socket.emit(...) その1人の Client に送る
    io.emit(...) 接続しているすべての Client に送る
    socket.on(...) イベントを受け取る

    たとえば、Server から今接続した Client だけにメッセージを送る場合は、次のように書く。

    socket.emit('welcome', 'Welcome!');

    接続している全員にメッセージを送る場合は、次のように書く。

    io.emit('receive_message', 'Hello everyone!');

    4. Step 1: Server から React Client にメッセージを送る¶

    4.1 Step 1 の目的¶

    Step 1 では、React Client が Server に接続したときに、Server が welcome メッセージを送る。

    流れは次の通りである。

    React Client が Server に接続する
        ↓
Server が connection を検出する
        ↓
Server が welcome イベントを送る
        ↓
React Client が welcome イベントを受け取る
        ↓
React Client が画面にメッセージを表示する

}

    5. Step 2: React Client から Server にメッセージを送る¶

    5.1 Step 2 の目的¶

    Step 2 では、ボタンをクリックしたときに、React Client から Server にメッセージを送る。

    その後、Server は同じ Client に返信する。

    流れは次の通りである。

    User がボタンをクリックする
        ↓
React Client が hello イベントを送る
        ↓
Server が hello イベントを受け取る
        ↓
Server が response イベントを返す
        ↓
React Client が response を画面に表示する

    6. Step 3: 匿名チャットを作成する¶

    6.1 Step 3 の目的¶

    Step 3 では、テキストボックスに入力したメッセージを Server に送り、Server がすべての Client に送信する。

    この Step では、送信者の名前は表示しない。

    そのため、匿名チャットである。

    流れは次の通りである。

    User がメッセージを入力する
        ↓
React Client が send_message イベントを送る
        ↓
Server が send_message を受け取る
        ↓
Server が receive_message を全 Client に送る
        ↓
すべての Client がメッセージを表示する

    6.2 server.js のコード¶

    chatapp/server/server.js を以下のコードに置き換える。

    const express = require('express');
const http = require('http');
const { Server } = require('socket.io');

const app = express();
const server = http.createServer(app);

const io = new Server(server, {
  cors: {
    origin: '*'
  }
});

io.on('connection', (socket) => {
  console.log('A user connected:', socket.id);

  socket.emit(
    'welcome',
    'Welcome! This is your first Socket.IO message.'
  );

  socket.on('hello', () => {
    console.log(`User ${socket.id} said hello`);

    socket.emit(
      'response',
      'Hi! Server got your hello.'
    );
  });

  socket.on('send_message', (msg) => {
    console.log(`User ${socket.id} says:`, msg);

    io.emit(
      'receive_message',
      msg
    );
  });
});

const PORT = 3001;

server.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});

    6.3 追加された Server 側のコード¶

    Step 3 で新しく追加された部分は次である。

    socket.on('send_message', (msg) => {
  console.log(`User ${socket.id} says:`, msg);

  io.emit(
    'receive_message',
    msg
  );
});

    socket.on('send_message', ...) は、Client から送られてくるチャットメッセージを受け取る処理である。

    msg には、Client が送ったメッセージが入る。

    次のコードで、接続しているすべての Client にメッセージを送る。

    io.emit(
  'receive_message',
  msg
);

    ここでは socket.emit ではなく io.emit を使う。

    Code Meaning
    socket.emit 1人の Client に送る
    io.emit すべての Client に送る

    6.4 App.js のコード¶

    chatapp/client/src/App.js を以下のコードに置き換える。

    import React, { useEffect, useState } from 'react';
import { io } from 'socket.io-client';

const socket = io('http://localhost:3001');

function App() {
  const [msg, setMsg] = useState('');
  const [reply, setReply] = useState('');
  const [chatLog, setChatLog] = useState([]);
  const [input, setInput] = useState('');

  useEffect(() => {
    socket.on('welcome', (data) => {
      setMsg(data);
    });

    socket.on('response', (data) => {
      setReply(data);
    });

    socket.on('receive_message', (data) => {
      setChatLog((prev) => [...prev, data]);
    });

    return () => {
      socket.off('welcome');
      socket.off('response');
      socket.off('receive_message');
    };
  }, []);

  const handleClick = () => {
    socket.emit('hello');
  };

  const sendMessage = () => {
    if (input.trim() !== '') {
      socket.emit('send_message', input);
      setInput('');
    }
  };

  return (
    <div style={{ padding: '20px', fontFamily: 'Arial' }}>
      <h2>Socket.IO Chat App: Step 3</h2>

      <p>
        <strong>Server says on connect:</strong> {msg}
      </p>

      <button onClick={handleClick}>
        Send Hello
      </button>

      <p>
        <strong>Server replied:</strong> {reply}
      </p>

      <hr />

      <h3>Anonymous Chat</h3>

      <input
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="Type a message..."
      />

      <button onClick={sendMessage}>
        Send
      </button>

      <ul>
        {chatLog.map((item, idx) => (
          <li key={idx}>{item}</li>
        ))}
      </ul>
    </div>
  );
}

export default App;

    テキストボックスに入力された文字を保存するための state である。

    Server から receive_message イベントを受け取ったときに実行される。

    setChatLog((prev) => [...prev, data]) は、以前のメッセージ一覧に新しいメッセージを追加する処理である。

    const sendMessage = () => {
  if (input.trim() !== '') {
    socket.emit('send_message', input);

  }
};

    この関数は、入力されたメッセージを Server に送る。

    input.trim() !== '' は、空のメッセージを送らないための確認である。

    <input

  onChange={(e) => setInput(e.target.value)}
  placeholder="Type a message..."
/>

    メッセージ入力用のテキストボックスである。

    <ul>
  {chatLog.map((item, idx) => (
    <li key={idx}>{item}</li>
  ))}
</ul>

    chatLog に入っているメッセージを一覧表示する。

    7. Step 4: 名前付きチャットを作成する¶

    7.1 Step 4 の目的¶

    Step 4 では、メッセージに送信者の名前を表示する。

    例：

    Alice: Hello
Bob: Hi Alice
Alice: Nice to meet you

    この Step が Mini-kadai 10 の提出対象である。

    7.2 Step 4 の流れ¶

    処理の流れは次の通りである。

    User が名前を入力する
        ↓
User がメッセージを入力する
        ↓
React Client が name と message を Server に送る
        ↓
Server が name と message を組み合わせる
        ↓
Server が全 Client に送る
        ↓
すべての Client が名前付きメッセージを表示する

    7.3 server.js のコード¶

    chatapp/server/server.js を以下のコードに置き換える。

    const express = require('express');
const http = require('http');
const { Server } = require('socket.io');

const app = express();
const server = http.createServer(app);

const io = new Server(server, {
  cors: {
    origin: '*'
  }
});

io.on('connection', (socket) => {
  console.log('A user connected:', socket.id);

  socket.emit(
    'welcome',
    'Welcome! This is your first Socket.IO message.'
  );

    console.log(`User ${socket.id} said hello`);

    socket.emit(
      'response',
      'Hi! Server got your hello.'
    );
  });

  socket.on('send_message', ({ name, message }) => {

    );

  console.log(`Server running on port ${PORT}`);
});

    Step 3 では、Server は1つの文字列を受け取っていた。

    Step 4 では、Server は object を受け取る。

    socket.on('send_message', ({ name, message }) => {

    React から送られる object は次のような形である。

    {
  name: "Alice",
  message: "Hello"
}

    Server では、次のように名前とメッセージを1つの文字列にする。

    const fullMessage = `${name}: ${message}`;

    結果は次のようになる。

    Alice: Hello

    そして、全 Client に送信する。

  'receive_message',

);

const socket = io('http://localhost:3001');

function App() {
  const [msg, setMsg] = useState('');
  const [reply, setReply] = useState('');
  const [chatLog, setChatLog] = useState([]);
  const [input, setInput] = useState('');
  const [name, setName] = useState('');

  useEffect(() => {
    socket.on('welcome', (data) => {
      setMsg(data);
    });

    socket.on('response', (data) => {
      setReply(data);
    });

    socket.on('receive_message', (data) => {
      setChatLog((prev) => [...prev, data]);
    });

    return () => {
      socket.off('welcome');
      socket.off('response');
      socket.off('receive_message');
    };
  }, []);

  const handleClick = () => {
    socket.emit('hello');
  };

  const sendMessage = () => {
    if (name.trim() !== '' && input.trim() !== '') {
      socket.emit(
        'send_message',
        {
          name: name,
          message: input
        }
      );

      setInput('');
    }
  };

  return (
    <div style={{ padding: '20px', fontFamily: 'Arial' }}>
      <h2>Socket.IO Chat App: Step 4</h2>

      <p>
        <strong>Server says on connect:</strong> {msg}
      </p>

      <button onClick={handleClick}>
        Send Hello
      </button>

      <p>
        <strong>Server replied:</strong> {reply}
      </p>

      <hr />

      <h3>Named Chat</h3>

      <div>
        <input
          value={name}
          onChange={(e) => setName(e.target.value)}
          placeholder="Enter your name"
        />
      </div>

      <br />

      <div>
        <input
          value={input}
          onChange={(e) => setInput(e.target.value)}
          placeholder="Type a message..."
        />

        <button onClick={sendMessage}>
          Send
        </button>
      </div>

      <ul>
        {chatLog.map((item, idx) => (
          <li key={idx}>{item}</li>
        ))}
      </ul>
    </div>
  );
}

export default App;

    7.6 Step 4 の React 側のポイント¶

    Step 4 では、名前を保存するための state を追加する。

    const [name, setName] = useState('');

    名前入力用のテキストボックスを追加する。

    <input
  value={name}
  onChange={(e) => setName(e.target.value)}
  placeholder="Enter your name"
/>

    メッセージを送る前に、名前とメッセージの両方が入力されているか確認する。

    if (name.trim() !== '' && input.trim() !== '') {

    Server に送るデータを object にする。

    socket.emit(
  'send_message',
  {
    name: name,
    message: input
  }
);

    これは次のようなデータを送っているという意味である。

    {
  name: "Alice",
  message: "Hello"
}

    短い書き方

    次のコードは、

    {
  name: name,
  message: input
}

    次のように短く書くこともできる。

    {
  name,
  message: input
}

    ただし、この授業では最初は分かりやすさを優先して、長い書き方を使っている。

    9. よくあるエラーと確認ポイント¶

    9.1 node server.js が動かない¶

    現在いる場所を確認する。

    正しい場所は次である。

    chatapp/server/

    正しいコマンドは次である。

    node server.js

    もし chatapp/ にいる場合は、先に次を入力する。

    cd server

    9.2 socket.io-client が見つからない¶

    React 側で socket.io-client がインストールされていない可能性がある。

    chatapp/client に移動する。

    cd chatapp/client

    次のコマンドを実行する。

    npm install socket.io-client

    その後、React を再起動する。

    npm start

    9.3 server.js を変更したのに反映されない¶

    server.js を変更した場合は、Server の再起動が必要である。

    Server の Terminal で次を押す。

    Ctrl + C

    その後、次を実行する。

    node server.js

    9.4 ブラウザが自動で開かない¶

    次の URL を手動で開く。

    http://localhost:3000

    9.5 チャットが2つのブラウザに表示されない¶

    次を確認する。

    • Server が起動しているか
    • React Client が起動しているか
    • 両方のブラウザで http://localhost:3000 を開いているか
    • server.js のチャット送信で io.emit を使っているか
    • server.js を変更したあと、Server を再起動したか
