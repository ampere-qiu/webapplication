## :material-chevron-right:課題1：リストでスタックを作る
リストクラスの`append`メソッドと`pop`メソッドを用いて、`push`メソッドと`pop`メソッド、`display`メソッドを備えた`ListStack`クラスを実装せよ。

今回作成する`ListStack`クラスの仕様は、以下の通りである

!!! example "ListStackクラス"
    **`#!python class ListStack()`**  
    スタックのクラス

    **インスタンス変数**

    - **list**: データ格納用のリスト

    **`#!python push(data)`**  
    スタックにデータをプッシュするメソッド

    **パラメータ**

    - **data**: スタックに挿入するデータ（`Stop`インスタンス）

    **`#!python pop()`**  
    スタックからデータをポップするメソッド

    **返り値**

    - スタックの先頭に格納されていたデータ（`Stop`インスタンス）。スタックが空であれば`None`を返す。

    **`#!python display()`**  
    スタックの中身を表示するメソッド

さらに、`kyotocitybus_stop.dat`からバス停データを読み込み、読み込んだ順に全てのデータをスタックにプッシュし、その後ポップしたデータを全て表示するよう、以下のプログラムの穴あき部分を埋めてプログラムを完成させよ。また、実行例の通りに動くことを確認し、実行結果がデータファイルと逆順になっていることに注意せよ。

## :material-chevron-right:発展課題4：継承やコンポジションを使う
[課題３](#3)で用いた`Stack`クラス、`ListStack`クラス、`Queue`クラス、`ListQueue`クラスにはコードの重複が多い。そこで、クラスの継承やコンポジションを用いてできる限りコードを再利用するように実装せよ。

!!! hint "ヒント"
    あるクラスのメソッドやインスタンス変数全てを引き継いで拡張したクラスを作ることを**継承**と呼ぶ。また、元のクラスを**親クラス**、継承したクラスのことを**子クラス**と呼ぶ。各スタックの実装やキューの実装に共通するメソッドを括りだし、親クラスを定義するとよい。一方、**コンポジション**と は、あるクラスで別のクラスのインスタンスを取り込んでその機能を使うことをいう。連結リスト版のスタックとキューで連結リストの同じ機能（たとえば先頭の節の切り取り等）を使うのであれば、その機能をもつ連結リストクラスを定義し、スタッククラスとキュークラスそれぞれで連結リストクラスのインスタンスを取り込むとよい。
